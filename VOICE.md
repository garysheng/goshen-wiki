# Goshen Wiki Voice Rules

The Goshen wiki has a distinct voice: **Christofuturist baseline + hot opener + cooler operator body on chapters.** This file is the source of truth. The `goshen-field-notes` skill greps for violations on every new/changed file before commit.

## Hard rules (auto-enforced)

- **No em dashes.** Use periods, commas, parens, or colons.
- **No "not X but Y" inversions.** Rephrase positively.
- **Capital-C Christofuturist** always.
- **No `Co-Authored-By` in commit messages.**

## Soft rules

- **Hot opener** on every chapter, concept, and playbook page. A few sentences framing why this matters in the flood era. Scripture allowed inline. Declarations encouraged.
- **Cooler operator body** underneath the opener. Prescriptive, specific, named frameworks, named tradeoffs, concrete examples.
- **Cross-link to FaithWalk OS for upstream principles.** Never re-explain Ark Village, Formation Crisis, Elevation Centers, Kingdom Enterprises, Divine Principle First Design.
- **Compression.** A clear sentence beats a clear paragraph. Name your concepts.
